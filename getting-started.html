Getting Started with Canon Protocol

This guide will walk you through setting up Canon Protocol and creating your first specification.

Installation

From Crates.io (Recommended)

# Install the latest version
cargo install canon-cli

# Verify installation
canon --version

Pre-built Binaries

# Linux x64
wget https://github.com/canon-protocol/canon-cli/releases/latest/download/canon-linux-x64
chmod +x canon-linux-x64

# macOS Apple Silicon
wget https://github.com/canon-protocol/canon-cli/releases/latest/download/canon-macos-apple
chmod +x canon-macos-apple

# Windows
# Download canon-windows.exe from the releases page

Quick Start

1 Initialize a Canon Repository 
canon init

This creates a canon.yml file with core dependencies and a .canon/ directory for cached specifications.

2 Install Dependencies 
canon install

Downloads and caches all specifications listed in your canon.yml file.

3 Add a Dependency 
canon add "api.io/openapi@2.0.0"

Adds a new dependency to your project and downloads it.

Creating Your First Specification

1. Define Your Type

Create a file my-spec.yml:

canon: 1.0
type: canon-protocol.org/type@1.0.0
name: my-api
version: 1.0.0
description: My first Canon specification

fields:
  - name: endpoint
    type: string
    required: true
    description: API endpoint URL
    
  - name: method
    type: string
    required: true
    enum: [GET, POST, PUT, DELETE]
    
  - name: headers
    type: map
    required: false
    description: HTTP headers

2. Create a Transformation

Add a transformation to handle natural language:

canon: 1.0
type: canon-protocol.org/transformation@1.0.0
name: my-api-transform
version: 1.0.0
target: my-publisher.com/my-api@1.0.0
priority: 500

patterns:
  - "call {method} on {endpoint}"
  - "make a {method} request to {endpoint}"
  - "{method} {endpoint} with {headers}"

transform: |
  function transform(input, matches) {
    return {
      endpoint: matches.endpoint,
      method: matches.method.toUpperCase(),
      headers: matches.headers || {}
    };
  }

Publishing to a Registry

Publishing requires setting up cryptographic keys for signing your specifications.

1. Initialize as a Publisher

canon publisher init my-publisher.com

This generates an Ed25519 key pair and creates necessary configuration.

2. Sign Your Specification

canon sign my-spec.yml

3. Publish to Registry

canon publish my-spec.yml --registry spec.farm

Understanding the Canon File Structure

my-project/
├── canon.yml                # Project configuration
├── .canon/                  # Cached dependencies
│   └── spec.farm/
│       └── specs/
│           └── canon-protocol.org/
│               ├── type/
│               │   └── 1.0.0/
│               └── transformation/
│                   └── 1.0.0/
└── specifications/          # Your specifications
    ├── my-api.yml
    └── my-transform.yml

Core Types

Canon Protocol provides these fundamental types:

  • type - Define new specification types
  • transformation - Transform natural language to structured data
  • auto-version - Automatically version specifications
